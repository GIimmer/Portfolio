Project 1: Draft Chat

C# on ASP.NET Core

This application allows several users across different computers to simulate a fantasy football draft in real time, using sockets, AJAX, complex DOM manipulation and real data.

Sockets: In order to have a great user experience that felt quick and intense, we used SignalR. This allows developers to create a 'hub' which keeps a persistant channel open between users, allowing live chat, instant updates and zero-delay turn switching.

AJAX: In building a one page application; we used AJAX for many of the functions on the page, in some cases querying the database for information and updating only one component on the players screen, in some cases triggering turn-end and updating everyone via AJAX+Sockets.

Complex DOM Manipulation: in order to reduce lag time and database use, we used custom front end code to allow users to sort the table - forward and reverse - by age, name, rank, etc. Being Merge Sort, this ran quickly with all 572 players in the db.

Project 2: User DashBoard

Python on Django

This application is a classic but well-built exercise in SQL relationships, user management and access. It allows many users to edit their profiles, leave messages, and comment on messages, it also implements user levels!

SQL elements: The three models of this application are Users, Messages, and comments. Users have information such as pass-hash, email, name, and description. Each user has a wall upon which messages can be left by themselves or others, then other users can leave comments on these messages. Users can delete only their own messages

Extras: Users build their basic profile upon signing in, then have the option to add their own description. User levels are implemented - Admins can edit and delete other users, whereas level 1 users cannot. Only level 9 admins can elevate other users' levels. Passwords are kept secure with BCrypt. Back end validation exist for all aspects (using session) and users can log in and out.
